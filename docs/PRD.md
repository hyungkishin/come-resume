# PRD: Foliofy — AI 포트폴리오 빌더 & 이력서 폴리셔

## 1. 한줄 요약

GitHub 연동하면 포트폴리오 사이트 자동 생성, 이력서 넣으면 AI가 깔끔하게 다듬어주는 올인원 커리어 툴.

## 2. 문제 정의

| 문제 | 현실 | 근거 |
|------|------|------|
| 포트폴리오 만들기 귀찮음 | 개발자 대다수가 포트폴리오 없이 이직함 | Stack Overflow 2023 Survey: 개발자 62%가 개인 사이트 없음 |
| 이력서 쓰기 어려움 | "뭘 어떻게 써야 하지?" — 자기 경험을 잘 못 표현함 | 블라인드/코드너리 설문: "이력서가 가장 어렵다" 응답 상위 |
| 디자인 못함 | 개발자가 만든 포트폴리오는 대체로 구림 | — |
| 업데이트 안 함 | 한번 만들고 방치 → 최신 프로젝트 반영 안 됨 | — |
| 비용 | 디자이너에게 포트폴리오 디자인+개발 의뢰 시 50~200만원 | 크몽/숨고 기준 평균 단가 |

## 3. 솔루션

**Foliofy** = GitHub 연동 자동 포트폴리오 + AI 이력서 폴리셔 + 프리미엄 디자인 템플릿

1. **GitHub 연결 → 포트폴리오 자동 생성** (5분)
2. **이력서 붙여넣기 → AI가 프로로 다듬어줌** (30초)
3. **한 클릭 배포** → username.foliofy.dev

## 5. 핵심 기능

### 5.1 MVP (v1.0)

#### 🔗 GitHub 연동 포트폴리오 자동 생성
- GitHub OAuth 로그인
- 레포지토리 분석: 언어, 스타, 커밋 빈도, README 파싱
- 핀한 레포 + 인기 레포 자동 선별
- 프로젝트 카드 자동 생성 (설명, 기술스택 태그, 스크린샷)
- 커밋 히트맵, 기술 스택 차트 자동 생성

#### 📝 AI 이력서 폴리셔
- 이력서 텍스트 붙여넣기 or PDF 업로드
- AI가 분석 후 개선:
  - **문장 다듬기**: 두루뭉술한 표현 → 임팩트 있는 성과 중심 문장
  - **STAR 포맷**: 상황-과제-행동-결과 구조로 재구성
  - **키워드 최적화**: JD(채용공고) 붙여넣으면 매칭되는 키워드 강화
  - **어색한 표현 교정**: 한글/영문 모두 지원
- Before/After 비교 뷰
- 섹션별 점수 + 구체적 개선 제안

#### 🎨 포트폴리오 에디터
- 드래그앤드롭 섹션 빌더
- 섹션 종류: 히어로, 소개, 프로젝트, 기술스택, 경력, 교육, 연락처
- 프리미엄 디자인 템플릿 10종+
- 실시간 미리보기 (데스크탑/모바일 전환)
- 커스텀 컬러/폰트/레이아웃

#### 🚀 원클릭 배포
- `username.foliofy.dev` 서브도메인 자동
- 커스텀 도메인 연결 (프리미엄)
- 자동 SSL
- SEO 최적화 (meta, OG, sitemap)
- 구글 애널리틱스 연동

#### 📄 이력서 PDF 익스포트
- 포트폴리오 데이터 기반 이력서 PDF 자동 생성
- 깔끔한 이력서 템플릿 5종+
- ATS(지원자추적시스템) 호환 포맷
- A4/Letter 사이즈 선택

### 5.2 엣지 케이스 & 예외 처리

| 시나리오 | 처리 방식 |
|----------|-----------|
| GitHub 레포 0개인 유저 | 수동 프로젝트 추가 모드로 안내, 빈 상태(empty state) UI |
| README 없는 레포 | 레포 설명(description) + 언어/스타 기반으로 카드 생성, "README 추가 권유" 배지 |
| 비공개 레포만 있는 유저 | GitHub 토큰 스코프로 private 접근 허용 (사용자 동의 필수), 비공개 표시 옵션 |
| 이력서가 아닌 텍스트 붙여넣기 | AI가 이력서 여부 판별 → 비이력서인 경우 안내 메시지 |
| PDF 파싱 실패 | 텍스트 직접 입력 폴백 안내, 스캔 이미지 PDF는 미지원 고지 |
| GitHub API rate limit 초과 | 큐잉 + 백오프 재시도, 사용자에게 "잠시 후 재시도" 안내 |

**Moat (지속 가능한 차별점)**:

## 11. 유저 플로우

### 플로우 1: 가입 → 포트폴리오 생성 → 배포

```
랜딩 페이지
  → "GitHub으로 시작하기" 클릭
  → GitHub OAuth 동의
  → 온보딩: 프로필 정보 확인 (이름, 직함, 한줄 소개)
  → 레포 선택 화면: 핀 레포 + 인기 레포 자동 추천, 체크박스로 추가/제거
  → 템플릿 선택 (Free 1종 / Pro 10종+)
  → 포트폴리오 자동 생성 (로딩 애니메이션, ~10초)
  → 에디터 진입: 실시간 미리보기 + 드래그앤드롭 편집
  → "배포하기" 클릭 → username.foliofy.dev 즉시 라이브
  → 배포 완료 화면: URL 복사, SNS 공유 버튼
```

### 플로우 2: 이력서 폴리싱

```
대시보드 → "이력서 폴리셔" 진입
  → 입력 방식 선택: 텍스트 붙여넣기 / PDF 업로드
  → (선택) JD 붙여넣기 → 매칭 키워드 강화 모드 활성화
  → AI 분석 시작 (로딩, ~5~15초)
  → 결과 화면:
      좌: 원본 (Before) / 우: 개선본 (After)
      섹션별 점수 (경력, 프로젝트, 기술스택 등)
      개선 사항 하이라이트 + 툴팁 설명
  → 섹션별 "적용/무시" 선택
  → 최종본 확인 → 복사 / PDF 다운로드 (Pro)
  → 사용 횟수 표시 (Free: 남은 횟수 / Pro: 무제한)
```

### 포트폴리오 배포 아키텍처

```
배포 방식: Vercel 와일드카드 도메인 + Next.js Dynamic Routes

*.foliofy.dev → Vercel 와일드카드 DNS
  → Next.js middleware에서 subdomain 추출
  → /portfolio/[username] 동적 라우트로 매핑
  → Supabase에서 포트폴리오 데이터 조회
  → ISR (Incremental Static Regeneration) 캐싱 (revalidate: 3600)

커스텀 도메인 (Pro):
  → Vercel Domains API로 도메인 추가
  → CNAME 검증 → 자동 SSL 프로비저닝
  → 도메인 → username 매핑 테이블
```

## 13. 보안 & 개인정보보호

### 인증 & 토큰 관리
- GitHub OAuth 토큰은 Supabase Auth에서 관리, 서버 사이드에서만 접근
- 토큰 스코프: `read:user`, `repo` (비공개 레포 선택 시에만 요청)
- 리프레시 토큰은 httpOnly 쿠키로 저장, 클라이언트 JS 접근 불가
- Supabase RLS (Row Level Security) 전 테이블 적용

### 이력서 데이터 보호
- 이력서 텍스트는 Supabase DB에 암호화 저장 (AES-256)
- AI 폴리싱 시 Claude API로 전송 → 요청/응답 로그 서버에 미저장
- 사용자 삭제 요청 시 30일 내 완전 삭제 (soft delete → hard delete)
- AI 폴리싱 이력은 최대 90일 보관 후 자동 삭제

### 법적 요구사항
- 개인정보처리방침: 수집 항목, 목적, 보관 기간, 제3자 제공 (Claude API) 명시
- 이용약관: 서비스 범위, 면책, 환불 정책
- GDPR 대응: 데이터 포터빌리티 (포트폴리오 JSON 익스포트), 삭제 요청 처리
- 한국 개인정보보호법: 개인정보 처리 위탁 동의 (Supabase, Anthropic)

### 계정 삭제 (오프보딩)
- 계정 삭제 시: 배포된 포트폴리오 48시간 내 비활성화
- 커스텀 도메인: DNS 설정 해제 안내 메일
- Stripe 구독: 즉시 취소, 잔여 기간 비례 환불 없음 (약관 명시)
- 이력서 데이터: 30일 유예 → 완전 삭제

### Phase별 의존 관계

```
Phase 0 (셋업)
  → Phase 1 (GitHub + 자동 생성) ← 핵심 데이터 모델 확정
    → Phase 2 (에디터) ← Phase 1의 데이터 구조 의존
    → Phase 3 (AI 폴리셔) ← Phase 1과 독립, 병행 가능
  → Phase 4 (배포) ← Phase 1, 2 완성 필요
    → Phase 5 (결제) ← Phase 4 이후
      → Phase 6 (런칭) ← 전체 완성
```

> **참고**: 템플릿은 MVP에서 3종으로 축소하고, 런칭 후 점진적으로 10종까지 확대. 이것만으로도 14주는 빠듯하며, 예비 기간 2주를 추가로 확보하는 것을 권장 (총 16주).
